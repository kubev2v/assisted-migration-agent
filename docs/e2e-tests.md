# E2E Tests

## Running

```bash
# Build and run (container mode, default)
make e2e

# Or explicitly
make e2e.container

# VM mode (externally managed infra)
make e2e.vm

# Clean up leftover containers
make e2e.container.clean
```

Key flags passed to the binary:

| Flag | Default | Description |
|------|---------|-------------|
| `-infra-mode` | `container` | `container` (Podman) or `vm` (external) |
| `-agent-image` | `$(IMAGE_NAME):$(IMAGE_TAG)` | Agent container image |
| `-backend-image` | `quay.io/kubev2v/migration-planner-api:latest` | Backend container image |
| `-vcsim-model-path` | `test/e2e/testdata/vcsim-model` | Path to vcsim XML model directory |
| `-iso-path` | `.` | Directory containing `rhcos-live-iso.x86_64.iso` |
| `-agent-api-url` | `https://localhost:8000` | Agent local API URL |
| `-agent-proxy-url` | `http://localhost:8080` | Agent proxy URL |

## Test infrastructure

All infrastructure is managed via the `InfraManager` interface (`test/e2e/infra/infra.go`).
In container mode, Podman orchestrates all components.

| Component | Container name | Port | Purpose |
|-----------|---------------|------|---------|
| PostgreSQL | `test-planner-db` | 5432 | Agent-local database for collected VMs |
| vcsim | `test-vcsim` | 8989 | VMware vCenter simulator loaded with XML model |
| Agent | `test-planner-agent` | 8000 (HTTPS) | The agent under test |
| Backend | `test-planner` | 7443 (agent), 3443 (user) | Migration Planner API (connected mode tests) |
| OIDC server | (in-process) | 9090 | Token issuer for backend auth |
| Agent proxy | (in-process) | 8080 | Observable reverse proxy to intercept agent-to-backend traffic |
| OIDC proxy | (in-process) | 8082 | Proxy between backend and OIDC server |

vcsim credentials: username `core`, password `123456`, URL `https://localhost:8989/sdk`.

## vcsim model

The model is a directory of XML files (`test/e2e/testdata/vcsim-model/`) loaded by vcsim
via the `-load` flag. It contains pre-baked `ObjectContent` XML for every managed object.

## Test catalog

### `tests.go` -- Agent e2e tests

Top-level `Describe("Agent e2e tests", Ordered)`. Postgres starts in `BeforeAll` and stops in `AfterAll`.

#### Disconnected env

| Context | Test | Description |
|---------|------|-------------|
| mode at startup | should not make requests to console when starting in disconnected mode | Starts agent in disconnected mode, waits 5s, asserts zero requests via proxy |
| mode at startup | should make requests to console when starting in connected mode | Starts agent in connected mode, waits 5s, asserts requests were observed |
| mode switching | should stop making requests after switching from connected to disconnected mode | Starts connected, observes requests, switches to disconnected, asserts no new requests |
| mode switching | should start making requests after switching from disconnected to connected mode | Starts disconnected, asserts no requests, switches to connected, asserts requests appear |
| mode switching | should persist mode after agent restart | Starts connected, switches to disconnected, restarts agent, asserts mode is still disconnected |
| collector | should collect inventory successfully with valid credentials | Starts collector with valid vcsim creds, waits for `collected` status, asserts inventory is non-nil |
| collector | should report error with bad credentials | Starts collector with invalid creds, asserts status reaches `error` |
| collector | should report error with bad vCenter URL | Starts collector with unreachable URL, asserts status reaches `error` |
| collector | should recover from bad credentials to successful collection | Fails with bad creds, retries with valid creds, asserts `collected` status |
| collector | should persist collected state and inventory after agent restart | Collects, restarts agent, asserts status is still `collected` and inventory is available |

#### Connected env

| Context | Test | Description |
|---------|------|-------------|
| mode at startup | should register agent with backend when starting in connected mode | Starts connected agent, asserts source has agent attached with `waiting-for-credentials` status |
| mode switch | should register agent with backend after switching from disconnected to connected | Starts disconnected, switches to connected, asserts agent appears on source |
| mode switch | should communicate with backend without errors when no inventory is collected | Starts disconnected, switches to connected without collecting, asserts no error in agent status |
| collector | should push inventory to backend after successful collection | Collects in connected mode, asserts source has inventory with vcenter_id |
| collector | should manually upload inventory from disconnected agent to backend | Collects in disconnected mode, uploads via API, asserts source has matching vcenter_id |
| collector | should report error with statusInfo on backend for bad credentials | Sends bad creds in connected mode, asserts backend source shows `error` status with `invalid credentials` message |
| collector | should preserve inventory on backend after agent restart | Collects, restarts agent, asserts inventory persists on backend |

### `test_vms_endpoint.go` -- VM endpoint e2e tests

Top-level `Describe("VM endpoint e2e tests", Ordered)`. Manages its own postgres, vcsim, agent, and collector
lifecycle via `BeforeAll`/`AfterAll`. All 50 VMs are collected once before any test runs.

#### Basic tests

| Test | Description |
|------|-------------|
| should list all 50 VMs | Lists all VMs with pageSize=100, asserts total=50 and len(vms)=50 |
| should get VM details by ID | Fetches first VM by ID, asserts non-empty name, memory > 0, cpuCount > 0 |
| should return error for non-existent VM | GETs a non-existent VM ID, asserts "not found" error |

#### Memory filters

| Test | Filter | Expected |
|------|--------|----------|
| should filter by memory minimum | memorySizeMin=32768 | 24 VMs (32+64+128 GB tiers: 8+8+8) |
| should filter by memory maximum | memorySizeMax=16384 | 18 VMs (4+8 GB tiers: 9+9) |
| should filter by exact memory tier | memorySizeMin=8192, memorySizeMax=8193 | 9 VMs (exactly 8 GB) |
| should filter by memory range spanning multiple tiers | memorySizeMin=16384, memorySizeMax=65537 | 24 VMs (16+32+64 GB: 8+8+8) |

#### Disk filters

| Test | Filter | Expected |
|------|--------|----------|
| should filter by disk minimum | diskSizeMin=512000 | >= 10 VMs with total disk >= 500 GB |
| should filter by disk maximum | diskSizeMax=204800 | >= 5 VMs with total disk < 200 GB |
| should filter by disk range | diskSizeMin=204800, diskSizeMax=512001 | >= 5 VMs in [200, 500) GB range |

#### Cluster and status filters

| Test | Filter | Expected |
|------|--------|----------|
| should filter by cluster | clusters=\<cluster name from first VM\> | 25 VMs (even/odd host split across 2 clusters) |
| should filter by status poweredOn | status=poweredOn | 50 VMs (all poweredOn) |
| should return empty for non-matching status | status=poweredOff | 0 VMs |

#### Combined filters

| Test | Filters | Expected |
|------|---------|----------|
| should combine memory min and disk min | memorySizeMin=32768, diskSizeMin=307200 | >= 5 VMs satisfying both |
| should combine memory range and disk range | memorySizeMin=8192, memorySizeMax=32769, diskSizeMin=204800, diskSizeMax=614401 | >= 1 VM in intersection |
| should combine memory filter with disk sort | memorySizeMin=8192, sort=diskSize:desc | 41 VMs, disk descending order |
| should combine memory filter, disk filter, sort, and pagination | memorySizeMin=8192, diskSizeMin=204800, sort=memory:desc, page=1, pageSize=5 | 5 VMs, correct totals, memory desc |
| should apply all filter dimensions together | memorySizeMin=16384, diskSizeMin=102400, status=poweredOn, clusters=\<name\> | >= 10 VMs matching all four |

#### Sorting

| Test | Sort | Assertion |
|------|------|-----------|
| should sort by name ascending | name:asc | Alphabetical order |
| should sort by name descending | name:desc | Reverse alphabetical |
| should sort by memory descending | memory:desc | Decreasing memory, all 50 VMs |
| should sort by diskSize ascending | diskSize:asc | Increasing disk, all 50 VMs |
| should apply multi-sort memory desc then name asc | memory:desc, name:asc | Primary: memory desc, secondary: name asc within same tier |

#### Pagination

| Test | Params | Assertion |
|------|--------|-----------|
| should paginate with correct metadata | page=1, pageSize=3 | 3 VMs, total=50, pageCount=17 |
| should return different VMs on different pages | page 1 vs page 2, pageSize=3 | No overlapping VM IDs |
| should return correct remainder on last page | page=17, pageSize=3 | 2 VMs (50 - 16*3) |
| should use default page size when not specified | no params | 20 VMs, total=50, pageCount=3 |
| should paginate filtered results correctly | memorySizeMin=131072, memorySizeMax=131073, page=1, pageSize=3 | total=8, pageCount=3, 3 VMs returned |
